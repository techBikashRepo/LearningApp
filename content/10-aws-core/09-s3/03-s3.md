## SECTION 9 — Certification Focus (AWS SAA-C03)

```
WHAT GOES IN S3?
  ✓ User uploads (images, videos, documents, files)
  ✓ Application artifacts (builds, deployments, Lambda ZIPs)
  ✓ Static website assets (HTML, CSS, JS, images)
  ✓ ML training datasets (large files, infrequent large-batch reads)
  ✓ Data lake raw data (CSV, Parquet, JSON for Athena/Glue processing)
  ✓ Backups and snapshots (RDS snapshots exported, EBS snapshots to S3)
  ✓ Log archives (ALB access logs, CloudTrail, VPC Flow Logs)

WHAT DOESN'T GO IN S3?
  ✗ Database (use RDS/DynamoDB — S3 is object store, not queryable like DB)
  ✗ Frequently mutated small files (EBS or EFS for file system semantics)
  ✗ Streaming real-time data (Kinesis for streams, S3 as destination after buffering)
  ✗ Shared file system between EC2 (use EFS for NFS-mount across EC2)

S3 vs EBS vs EFS:
  S3:  Object storage. REST API. Any size. Cheapest. Not mountable on OS.
  EBS: Block storage. Attached to 1 EC2 instance. Fastest I/O. AZ-locked.
  EFS: Network file system. Mountable on multiple EC2 simultaneously. NFS protocol.

  Choose S3: file sharing across services/regions, web-accessible, any scale
  Choose EBS: single EC2 needs fast I/O (DB, swap, OS)
  Choose EFS: multiple EC2 need same file system (CMS, shared config, legacy apps)
```

## SECTION 10 — Comparison Table

```
ACCESS FREQUENCY → STORAGE CLASS SELECTION:

Multiple times daily → S3 Standard ($0.023/GB)
  Example: production image serving, CDN origin files, active database backups

Once per month → S3 Standard-IA ($0.0125/GB + retrieval fee $0.01/GB)
  Example: monthly report archives, old deployment artifacts
  Note: min 30-day storage charge. Small frequent objects cost more on IA.

Once per quarter → S3 Glacier Instant Retrieval ($0.004/GB)
  Example: quarterly compliance reports, seasonal data
  Millisecond retrieval (unlike old Glacier)

Once per year or less → S3 Glacier Flexible ($0.0036/GB)
  Example: annual audit archives, legal hold data
  Retrieval: expedited (1-5 min), standard (3-5 hr), bulk (5-12 hr)

7+ year compliance hold → S3 Glacier Deep Archive ($0.00099/GB)
  Example: financial records (7-year retention), HIPAA records
  Retrieval: 12 hours minimum

Unknown/unpredictable → S3 Intelligent-Tiering ($0.0025/1K objects monitoring)
  Automatically moves objects between Standard, IA, Archive tiers
  Zero retrieval fee (unlike standard IA)
  Best for: data with unpredictable or changing access patterns

LIFECYCLE RULE TEMPLATE:
  Standard → Standard-IA: after 30 days
  Standard-IA → Glacier Instant: after 90 days
  Glacier Instant → Deep Archive: after 365 days
  Delete: after 2555 days (7 years)
  Non-current versions: delete after 90 days
```

## SECTION 11 — Quick Revision

```
TRAP 1: S3 is NOT a file system — no POSIX operations
  Cannot: rename (must copy + delete original)
  Cannot: partial update (must re-upload entire object)
  Cannot: atomic move across buckets
  Exam: "application needs atomic file rename" → S3 cannot do this → EFS or EBS

TRAP 2: S3 bucket names are globally unique across ALL accounts
  "my-bucket" taken by ANY account in any region → unavailable to you
  Exam: "cannot create bucket named 'logs'" → likely already exists globally

TRAP 3: S3 Strong Consistency (since Dec 2020)
  Old behavior: eventual consistency (reads after writes might see old data)
  Current: strong read-after-write consistency for all operations
  Exam: if question references "eventual consistency in S3" → outdated, now strongly consistent

TRAP 4: S3 Transfer Acceleration ≠ replication
  Transfer Acceleration: uploads via CloudFront edge → faster for distant clients
  CRR (Cross-Region Replication): async copy of objects to another Region bucket
  Exam: "user in Australia uploading to us-east-1 bucket slowly" → Transfer Acceleration
  Exam: "need data backed up to eu-west-1 automatically" → CRR

TRAP 5: Glacier retrieval has minimum storage duration charges
  Glacier Instant: minimum 90-day storage charge per object
  Glacier Flexible: minimum 90-day charge
  Glacier Deep Archive: minimum 180-day charge
  Exam: "store objects in Deep Archive for 30 days then delete" → still billed 180 days

TRAP 6: S3 static website hosting uses HTTP, not HTTPS
  S3 static website endpoint: http://bucket.s3-website.region.amazonaws.com (HTTP only)
  For HTTPS: use CloudFront in front of S3 (CloudFront adds SSL termination)
  Exam: "static website needs HTTPS" → CloudFront + S3 origin (not S3 website hosting alone)
```

## SECTION 12 — Architect Thinking Exercise

```
REQUIREMENT:
  SaaS company: 50,000 customers upload documents (contracts, invoices, reports).
  Requirements: versioning, compliance retention (7 years), secure access per customer.

SOLUTION DESIGN:

  1. BUCKET STRUCTURE
     Option A: 1 bucket, prefix per customer (s3://docs-prod/tenant-{id}/...)
     Option B: 1 bucket per customer (50,000 buckets → management complexity)
     Decision: 1 bucket, prefix isolation, bucket policy with condition on prefix

  2. ACCESS CONTROL
     IAM role per service: only app servers can read/write
     Pre-signed URLs: generate for customer downloads (15-minute expiry)
     No public access: Block Public Access enabled at account level

  3. VERSIONING + PROTECTION
     Versioning: enabled (recover accidental deletions/overwrites)
     Object Lock: COMPLIANCE mode for regulated documents (cannot delete for 7 years)
     MFA Delete: require MFA to delete version (extra protection)

  4. LIFECYCLE POLICY
     Current version Days 0-365: S3 Standard
     Current version Day 365+: S3 Standard-IA (less frequent access after 1 year)
     Non-current versions: Standard 30 days → Glacier Deep Archive → expire at 7 years

  5. REPLICATION
     CRR: replicate to secondary region (DR, 99.999999999% × 99.999999999%)

  6. MONITORING
     S3 Storage Lens: visibility across all prefixes
     CloudWatch: NumberOfObjects, BucketSizeBytes (alert if > $X/month)
     S3 Server Access Logging: audit trail of all object access

COST ESTIMATE (50K customers × 100MB average = 5TB active):
  Standard: 5TB × $0.023 = $115/month
  IA (archived docs): 20TB × $0.0125 = $250/month
  Deep Archive (compliance): 100TB × $0.00099 = $99/month
  Requests: ~$50/month
  Total: ~$514/month for petabyte-class compliance system
```

### Architect's Mental Model

```
5 RULES I NEVER VIOLATE:

1. "S3 is the right answer for any file that lives beyond the lifetime of a service"
   EC2 terminated, Lambda completed, ECS task stopped → S3 data remains.
   Persistent files belong in S3, not on ephemeral compute storage.

2. "Never make an S3 bucket public unless it's exclusively serving static assets via CDN"
   CloudFront + OAC + private S3 = more secure AND cheaper than public S3.
   All user data S3 buckets: private. Always. No exceptions.

3. "Versioning + lifecycle rules must be set together — never one without the other"
   Versioning without lifecycle = unbounded growing cost.
   Set lifecycle rule on versioning DAY ONE. Never retrofit later into existing debt.

4. "CloudFront in front of S3 is the default — not the exception"
   Direct S3 URL: high latency for global users, high egress cost, no SSL customization.
   CloudFront: SSL (custom domain), edge cache (reduce latency 80%), no egress charge from S3.

5. "S3 storage class is set per object — intelligent-tiering is the safe default for unknowns"
   Don't guess access patterns. Use Intelligent-Tiering for uncertain data.
   Cost: slightly higher monitoring fee + no retrieval fees = correct for unpredictable access.

3 MISTAKES JUNIOR ARCHITECTS MAKE:

1. Using S3 as a shared file system between EC2 instances
   S3 lacks POSIX semantics. For shared file system: EFS.
   S3 for: read-once-serve, media, backups. NOT for mounted shared directories.

2. Forgetting egress costs from S3 → $0.09/GB is significant at scale
   Fix: CloudFront in front (S3→CloudFront = free). CloudFront→user = $0.0085/GB.
   Also: S3 VPC endpoint for EC2 → S3 transfers within same region = free.

3. Setting versioning without lifecycle rule on delete markers and old versions
   Delete markers accumulate. Old versions accumulate.
   Both cost money. Both require explicit lifecycle rules to clean up.

30-SECOND MENTAL MODEL (Say this in an interview):
  "S3 is an infinitely scalable object store — pay per byte stored and per request.
   11 nines durability, data spread across 3+ AZs, globally accessible via HTTPS.
   Default to S3 for any file that needs to outlive a compute instance.
   Security: all buckets private, CloudFront + OAC for public-facing assets.
   Cost: set lifecycle rules on day one. Versioning without lifecycle is a cost trap."
```
